How it works

  1. Type symptoms (free text, e.g. "headache with runny nose").
  2. Local-first — We match against a curated DB of 10 conditions and return rich, prewritten advice.
  3. Fallback — If not found, we display a clearly-labelled simulated API response (demo-only) and recommend consulting a doctor.

Architected to plug in real symptom APIs (Infermedica, etc.) once keys/approval are granted.

Demo-ready architecture
  • Streamlit / Flask frontend
  • Separable `conditions_db.py` for easy editing
  • `api_fallback.py` currently simulated
  • Designed to be production-ready for approved APIs
